# Code of Conduct

## Our Pledge

We want Gennety to be a focused, respectful, and useful open source project. Contributors, maintainers, and users are expected to participate in a way that keeps collaboration constructive and safe.

## Expected Behavior

- Be respectful and direct.
- Assume good intent, but prioritize clear technical reasoning.
- Keep feedback specific, actionable, and tied to the work.
- Respect user privacy, contributor privacy, and project security boundaries.
- Report security issues privately instead of opening public issues.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information, credentials, production access details, or sensitive user data.
- Deliberately bypassing project privacy rules or encouraging unsafe handling of user context.
- Repeatedly derailing issues or pull requests away from the stated topic.

## Enforcement

Maintainers may remove comments, close issues, block users, or restrict participation when behavior violates this Code of Conduct. Enforcement decisions should be proportional to the behavior and focused on preserving a healthy project environment.

## Reporting

Report conduct concerns to the maintainers through GitHub or by email when a private channel is needed. Security vulnerabilities must be reported according to [SECURITY.md](SECURITY.md).
